BC2Fabric Prerequisites

Review the infrastructure, access, and configuration requirements before installing the BC2Fabric workload in your Microsoft tenant.

Download the full installation guide (PDF)

1. Business Central environment readiness

  • Confirm you are running Dynamics 365 Business Central online (version 23 or later) in the production or sandbox environment where BC2Fabric will synchronize data.
  • Install the BC2Fabric extension from Microsoft AppSource and assign it to the companies that will publish telemetry and application data to Fabric.
  • Create a dedicated Business Central user (service account) with the BC2Fabric user group or equivalent permissions to read the target tables and system events.
  • Verify that the environment time zone, job queue frequency, and data retention meet the synchronization requirements outlined in the installation checklist.

2. Microsoft Entra ID application registration

  • Register a single-tenant application in Microsoft Entra ID to represent BC2Fabric and capture the application (client) ID and directory (tenant) ID.
  • Create a client secret that will be shared with the BC2Fabric workload and note its expiration date for future rotations.
  • Grant the application delegated permissions for Dynamics 365 Business Central, Azure Key Vault, and any auxiliary APIs required by your deployment. Admin consent is needed before installation.
  • Add the service account as an application user within Business Central, mapping it to the BC2Fabric permission set.

3. Azure Key Vault configuration

  • Provision an Azure Key Vault in the same tenant that hosts Business Central and Microsoft Fabric.
  • Store the BC2Fabric client secret and any integration credentials (for example, SQL or storage account keys) as individual secrets following a consistent naming convention.
  • Configure access policies so that the BC2Fabric managed identity and operations team can retrieve secrets while limiting write access to administrators.
  • Enable purge protection and soft-delete to safeguard credentials from accidental removal during lifecycle operations.

4. Microsoft Fabric capacity and workspace

  • Ensure your tenant has a Fabric capacity (F64 or higher is recommended for production) with available resources to run the BC2Fabric pipelines and lakehouse refreshes.
  • Create or select a dedicated workspace that will host the BC2Fabric assets. The workspace admin should match the owner of the Entra ID application.
  • Assign the BC2Fabric service principals and operations staff as workspace members with contributor or higher roles.
  • Validate data region compliance requirements when selecting the capacity and workspace locations.

5. Network and security prerequisites

  • Allow outbound connections from Business Central and Fabric to the BC2Fabric API endpoints and to Azure storage services used for staging.
  • Review conditional access policies and multifactor prompts that might block automated authentication for the BC2Fabric service accounts.
  • Document firewall exceptions for on-premises data gateways if they are part of your data export strategy.
  • Capture audit requirements and retention policies so the Fabric workspace can be configured with the correct data lake storage and monitoring integrations.

6. Deployment checklist

  • Gather tenant ID, application ID, client secret name, and key vault URI prior to starting the installer.
  • Confirm Business Central companies enabled for BC2Fabric have job queues running and that administrators can schedule updates during off-peak hours.
  • Ensure Fabric workspace roles, lakehouse storage accounts, and pipelines are named per your organization's governance policies.
  • Plan for post-install validation by identifying dashboards or SQL endpoints that will consume the synchronized Business Central data.
