Platform Architecture

Understand how BC2 Fabric components interact so you can design resilient solutions.

Core services

The platform is composed of orchestration, processing, and delivery services. Each service is containerized and deployed across multiple availability zones.

  • Ingestion Gateway: Handles inbound traffic, auth, and throttling.
  • Stream Processor: Normalizes, enriches, and routes events.
  • Delivery Engine: Dispatches events to downstream systems.

Data flow

  1. Clients connect via REST, gRPC, or WebSocket transports.
  2. Events are persisted to a multi-tenant queue for durability.
  3. Processing pipelines apply transformations and policy enforcement.
  4. Outputs are delivered via connectors defined in integration profiles.

Reliability

BC2 Fabric uses rolling deployments, automatic retries, and configurable DLQs to maintain availability under failure conditions.
